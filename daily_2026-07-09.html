UBEP: Re-architecting Expert Parallelism Communication Library for Production Superpods

想象你管一个有 100 台服务器的「奶茶店」，客人点单（token）需要分配给不同店员（expert），传统方式所有店员要等所有订单到齐才开工（BSP 同步），太慢了。MoE（混合专家）模型部署在 NVIDIA NVL72/576 跟华为 CloudMatrix384 这种「超级服务器」上时，通信就卡在三个瓶颈：(1) 同步等待浪费时间；(2) 高速网络也救不了同步开销；(3) 不同 expert 忙闲不均。UBEP 通过重新设计 All-to-All 通信库（店员之间互发订单的协议），让 All-to-All 延迟降低 52.4%，MoE 推理每个 token 输出时间降低 11.1%。

📖 专业版问题讲解 (展开)
We propose UBEP (Unified-Bus Expert Parallelism), a production-ready communication library that re-architects MoE All-to-All primitives for modern superpod architectures (NVIDIA NVL72/576, Huawei CloudMatrix384). It addresses three bottlenecks: coarse-grained BSP execution serialization, prohibitive synchronization overhead, and distance-agnostic scheduling. UBEP reduces All-to-All latency by up to 52.4% and MoE inference TPOT by up to 11.1%.
⚙️ 方法：Re-architects All-to-All primitives: unified-bus scheduler, distance-aware token routing, lightweight synchronization primitives replacing BSP.
⚖️ 取舍：Production-ready (not a research prototype) but tied to specific superpod topologies; benefits drop on non-hyperscaler clusters; communication library invasiveness requires framework integration effort.
📊 效果：Up to 52.4% All-to-All latency reduction; up to 11.1% MoE inference TPOT improvement; production-validated on NVIDIA NVL72/576 + Huawei CloudMatrix384.
🏢 团队：Yipeng Liu, Chang Liu, Si Shen, Jiaqi Zheng, Mingfan Li, Yuyang Yang, Guanhua Li, Yuquan Zhang, Yimeng Xu, Zhongzhe Hu, Zhiyuan Huang, Qihang Duan, Junsong Wang, Wenkai Ling, Baochuan Yang, Xianzhi Yu, Han Bao, Yijie Chen, Guihai Chen (Nanjing University + Huawei)

MatrixFSDP: Communication-Free Matrix Optimizers under ZeRO-3 Parameter Sharding

想象你训练一个超大的 AI 模型（参数存在很多张显卡上），传统方案是「用 FSDP/ZeRO-3 切分存储」（每人存一份切片），但 Muon 这种「高级优化器」需要看完整矩阵才能算（要整张图才能下一步），所以每一步都要先把切片拼回完整再切，浪费大量通信。MatrixFSDP 的巧妙之处：让一张显卡当「owner」存完整矩阵（其他人存空切片），反向传播自然把完整数据传到这张卡，Muon 算的时候就在本地算，不用每步拼图。结果：8 卡机子上 optimizer 步比 FSDP2-Muon 快 4.2×，8 节点 (64 卡) 快 54.6×。

📖 专业版问题讲解 (展开)
We present MatrixFSDP, an FSDP2 runtime for owner-shaped ZeRO-3 shards that enables whole-matrix optimizers (Muon, Shampoo, SOAP) without optimizer-step matrix communication. It uses a balance-aware owner planner, deterministic owner-segment P2P collectives, owner-buffer pinning, and owner-shard checkpoint resharding. On 64 A100s, reduces optimizer-step latency over FSDP2-Muon by 4.2× on one node and 54.6× on eight nodes, up to 2.15× end-to-end speedup.
⚙️ 方法：Owner-shaped ZeRO-3 placement (one rank owns full 2D matrix, others empty shards) + MatrixShard metadata + balance-aware owner planner + deterministic owner-segment P2P collectives + owner-buffer pinning + DCP checkpoint resharding.
⚖️ 取舍：Owner-shaped sharding breaks equal-size collective assumptions (custom P2P needed); forward/backward still materialize/reshard parameters (no savings on those phases); tensor-parallel-owned fragments out of scope.
📊 效果：4.2× / 54.6× optimizer-step speedup on 1/8 nodes (64 A100s); up to 2.15× end-to-end; preserves ZeRO-3 memory; matches full-matrix Muon trajectory on real-data loss; extends to Shampoo and SOAP.
🏢 团队：Ming Gao, Yanwu Xu, Hao Zhang (University of Pittsburgh + Google + Tsinghua University)

Energy-Efficient GPU DVFS for Fine-Tuning of SLMs on Resource-Constrained Embedded Devices

📖 专业版问题讲解 (展开)
We characterize fine-tuning behavior of representative encoder-only (BERT variants) and decoder-only (Pythia variants) SLMs on GLUE benchmarks under DVFS on resource-constrained embedded GPUs, identifying energy-efficient frequency/voltage policies that exploit the iterative forward-backward nature of fine-tuning (vs single-pass inference).
⚙️ 方法：Empirical characterization of DVFS on BERT + Pythia SLMs fine-tuning workloads; identifies batch-size and layer-aware frequency/voltage sweet spots; proposes energy-efficient scheduling policy.
⚖️ 取舍：Empirical (not theoretical); BERT/Pythia coverage may not generalize to all SLM architectures; DVFS gains are workload-specific; small model size limits absolute energy savings despite percentage improvements.

TileLens: Efficiently Using Large-Granularity Memory Systems with Transparent Two-Dimensional Memory Layout

想象 GPU 显存 (HBM) 就像一张「细网格的画布」, 但新一代「高带宽闪存」(HBF) 跟 RoMe 等「大颗粒存储」要求至少 4KB 一次访问 (一整块瓷砖那么大), 在 LLM 推理「分块矩阵乘法」里, 大颗粒访问会损失一个数量级性能 (10× 慢)。TileLens 提出「透明二维内存布局」: 不动用户代码, 在系统层把细粒度矩阵乘法重新组织成适合大颗粒存储的二维 tile 访问模式, 性能损失大幅缩小。

📖 专业版问题讲解 (展开)
We analyze why Large-Granularity Memory Systems (LGMS, like HBF and RoMe with 4KB minimum access granularity) degrade tiled matrix-multiplication (the dominant op in LLM inference) by up to an order of magnitude, and propose TileLens with a transparent two-dimensional memory layout that reorganizes tile accesses to match LGMS granularity, recovering most of the performance without user-code changes.
⚙️ 方法：Transparent 2D memory layout: system-layer reorganization of fine-grained GEMM tile accesses into LGMS-friendly 2D patterns; transparent to user code (compiler/runtime); supports both HBF and RoMe.
⚖️ 取舍：Transparent layer adds runtime/compilation overhead (smaller than naive LGMS penalty); 2D layout assumptions may not match all access patterns; HBF/RoMe not yet widely deployed so immediate benefit limited; co-design with hardware needed for full gain.
📊 效果：Recovers most of LGMS performance penalty (vs naive tiled GEMM); enables practical use of HBF/RoMe for LLM inference at scale; preserves HBM-scale programming model.
🏢 团队：Jae Hyung Ju, Euijun Chung, Hritvik Taneja, Anish Saxena, Shinnung Jeong, Hyesoon Kim, Moinuddin K. Qureshi (Georgia Institute of Technology)

FAPlace: Joint Optimization of Chiplet Placement and Interposer Footprint for 2.5D Systems

想象你把多个小芯片 (chiplet) 像拼图一样放在一张「硅中介层」(silicon interposer) 上做 2.5D 集成 (NVIDIA Blackwell / AMD MI300 / 各家 ASIC 厂都这么干), 传统方法是先定中介层大小再放芯片, 但「中介层多大」本身取决于「芯片怎么放」——鸡生蛋问题。FAPlace 第一次同时优化「芯片放哪」跟「中介层多大」, 解决「中介层太小放不下, 太大浪费」的两难, 对 2.5D 先进封装 (CoWoS / SoIC / EMIB) 时代至关重要。

📖 专业版问题讲解 (展开)
We present FAPlace, a placement approach that jointly optimizes chiplet placement and silicon interposer footprint for 2.5D system integration, breaking the circular dependency where the optimal interposer size depends on placement but placement is constrained by interposer dimensions.
⚙️ 方法：Joint optimization formulation: chiplet positions + interposer dimensions co-solved; handles undersized (infeasible) and oversized (sparse) interposer regimes; considers electrical/thermal constraints alongside geometric placement.
⚖️ 取舍：Joint optimization increases solver complexity (vs decoupled approach); assumes fixed chiplet set (no chiplet re-selection); manufacturing constraints (yield, defect) not fully modeled; thermal/electrical co-design simplified.
📊 效果：Avoids infeasible placements from undersized interposers; avoids waste from oversized interposers; produces denser, more cost-effective 2.5D layouts; applicable to CoWoS / SoIC / EMIB-class packaging.
🏢 团队：Yubo Hou, Sezin Kircali Ata, Gen Liang Lim, Richard Chang, Mihai Dragos Rotaru, Rahul Dutta, Ashish James (A*STAR Institute for Infocomm Research + A*STAR Institute of Microelectronics)
