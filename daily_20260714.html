STEEL: Sparsity-Aware Fused Attention for Energy-Efficient Long-Sequence Inference on AMD XDNA NPU

想象你买了个笔记本, 想让里面的 AI 助手快速处理长对话——但笔记本里那个 NPU (神经处理单元, 专为 AI 加速的小芯片) 跑长文本时会卡顿, 因为传统注意力算法没有针对 NPU 的特殊架构优化。AMD 跟 ETH 苏黎世 + 博洛尼亚大学团队做的 STEEL, 是首个开源的「针对 XDNA NPU 的 FlashAttention」实现。它把「预先读全文找关联 (prefill)」这部分重写成一种叫 dataflow 的并行执行方式, 充分利用 NPU 的空间并行 (即多计算单元同时跑); 同时针对「带因果遮罩 (causal mask, 让 AI 不能偷看未来词)」导致的负载不均, 用「感知稀疏 (sparsity-aware)」的流水线排布解决——简单说就是把不需要计算的位置跳过, 让 NPU 的工作更均衡。

📖 专业版问题讲解 (展开)
STEEL is the first open-source FlashAttention implementation targeting XDNA-like NPUs in laptop SoCs. It introduces a dataflow formulation of prefill attention that exploits spatial parallelism and on-chip memory, plus a sparsity-aware pipeline placement that mitigates causal-mask-induced load imbalance, reducing synchronization overhead and improving utilization on the NPU array.
⚙️ 方法：Dataflow-formulated prefill attention kernel + causal-mask sparsity-aware NPU pipeline placement, exported as open-source codebase.
⚖️ 取舍：Restricted to XDNA-like NPU architecture; benefits depend on sparsity pattern of attention workload; prefill-only optimization (decode path not covered).
📊 效果：On XDNA NPU, STEEL reaches higher energy efficiency than cloud-offloading baseline; first open FlashAttention implementation for XDNA-like NPUs.
🏢 团队：Victor J.B. Jung, Gagandeep Singh, Joseph Melber, Kristof Denolf, Francesco Conti, Luca Benini → AMD Research and Advanced Development (RAD); ETH Zürich; University of Bologna

StreamDQ: Near-Memory Weight DeQuantization in Custom HBM for Scalable AI Inference Acceleration

大模型越来越胖, 一个 700 亿参数的模型要占 140GB 显存, 通常会用「量化 (quantization)」压缩到 4-bit 或 8-bit——但还原 (dequantization) 时, GPU 的 CUDA 核心要去算, 反而成为瓶颈。SK Hynix 团队把「解量化」硬件直接做进了 HBM (高带宽内存) 的底层 (base die)——这是 HBM 内部的一个逻辑层, 跟 DRAM 堆叠分开, 原本只负责数据传输。StreamDQ 在 base die 里加了「解量化块 (Dequantization Block, DQB)」, 数据从 DRAM 出来时已经还原成 GPU 能直接用的格式, 省掉了 GPU 反复搬运 + 解码的开销。

📖 专业版问题讲解 (展开)
StreamDQ proposes lightweight DeQuantization Blocks (DQBs) embedded in the base die of HBM stacks, enabling on-the-fly weight-only dequantization. By relocating dequantization from GPU CUDA cores to the memory subsystem, StreamDQ eliminates instruction overhead, on-chip traffic, and pipeline stalls in weight-only quantization inference workloads at cloud scale.
⚙️ 方法：Embedded DeQuantization Blocks (DQB) in HBM base die, performing inline dequantization during data transfer to GPU.
⚖️ 取舍：Custom HBM stack requires SK Hynix base-die redesign; only benefits weight-only quantization inference (not training or non-quantized models).
📊 效果：Reduces on-chip traffic and pipeline stalls; accelerates high-throughput large-batch LLM serving in cloud-scale inference scenarios.
🏢 团队：Minki Jeong, Daegun Yoon, Soohong Ahn, Seungyong Lee, Nameun Kang, Hyeonseok Ju, Ieryung Park, Joonseop Sim, Youngpyo Joo, Hoshik Kim → SK Hynix

Attention to Detail: Evaluating Energy, Performance, and Accuracy Trade-offs Across vLLM Configurations

vLLM 是现在最流行的 LLM 推理引擎, 里面有很多「开关」可以调——比如用哪种注意力算子 (PagedAttention / FlashAttention / XFormers)、要不要开启 prefix caching (记住同一段对话开头的处理结果以避免重复算)、要不要分块做 prefill (把长输入切成小段逐段处理)。法国 Lille 大学 + 阿姆斯特丹自由大学 + 波尔多大学做了 9000 次实验跑 93,600 个测量点, 看这些开关怎么影响「耗电多少」「回答多快」「答案对不对」。结论是: 注意力类型和 prefix caching 对能耗影响最大, chunked prefill 影响最小; 不同任务的最优配置差别很大, 没有「一刀切」配置。

📖 专业版问题讲解 (展开)
A controlled study of 3 vLLM configuration options (attention kernel type, prefix caching, chunked prefill) across 5 open-weight LLMs and 5 inference tasks, totaling 9000 runs and 93600 measurements. Analyzes main effects and interactions of energy, latency, and accuracy.
⚙️ 方法：Full-factorial controlled experiment of 3 vLLM config options × 5 LLMs × 5 tasks = 9000 runs, 93600 measures.
⚖️ 取舍：Limited to 5 open-weight LLMs and 5 inference tasks; results may not generalize to larger models or production traffic patterns.
📊 效果：Attention kernel type and prefix caching dominate energy/performance trade-offs; chunked prefill has marginal impact; optimal config is task-dependent.
🏢 团队：Nada Zine, Tristan Coignion, Vincenzo Stoico, Ivano Malavolta, Clément Quinton, Romain Rouvoy, Patricia Lago → Univ. Lille, CNRS, Inria, IUF; Univ. Bordeaux, CNRS, LaBRI; Vrije Universiteit Amsterdam

EcoKube: Simulating Carbon-Aware Scheduling Policies in Heterogeneous Edge-Cloud Environments

AI 训练 + 推理现在耗电惊人, 全球数据中心年用电量已超 2%。问题是: 各地电网的「碳强度 (carbon intensity)」实时不同 (风电场白天碳强度低、火电厂晚上碳强度高), 数据中心 PUE (电能使用效率) 也参差不齐, 不同地点 + 不同时间把任务调度过去能省很多碳。阿姆斯特丹大学做了 EcoKube 仿真框架, 模拟「边缘 + 云」混合环境下的碳感知调度策略——简单说就是「碳排放低的地方 + 碳强度低的时间段优先跑任务」。它跟 Kubernetes 默认调度器 + KEIDS + TOPSIS/KCSS 三个基线策略对比, 提供一致的评估环境。

📖 专业版问题讲解 (展开)
EcoKube is a configurable simulation framework for reproducible evaluation of sustainability-aware scheduling policies in heterogeneous edge-cloud environments. It models time-varying grid carbon intensity, site PUE, and heterogeneous hardware, and is evaluated against Kubernetes default, KEIDS, and TOPSIS/KCSS baselines.
⚙️ 方法：Event-driven deterministic simulator + policy hooks + heterogeneity-aware reference policy (EcoKube).
⚖️ 取舍：Simulation-based (not production deployment); carbon models depend on grid + PUE assumptions that vary by region; does not capture dynamic hardware heterogeneity.
📊 效果：Enables reproducible comparison of carbon-aware schedulers; reference policy outperforms Kubernetes default, KEIDS, TOPSIS/KCSS on synthetic batch workloads.
🏢 团队：Gonçalo Ferreira, Shashikant Ilager → University of Amsterdam

Accelerating GPU Inference of Large Language Models with Moderately Unstructured Sparse Weight Matrices

想给大模型「剪枝 (pruning)」——就是把不重要的权重砍掉来加速推理, 但又想保持质量, 通常只能剪掉 50% 左右的「中等稀疏度」。问题是这个程度的稀疏, 现有的 GPU 稀疏矩阵乘法 (SpMM) 内核反而跑得比稠密矩阵还慢——因为稀疏数据的存储和访问开销抵消了计算节省。新加坡国立大学 + 浙大团队设计了一种三层矩阵存储格式: (1) 稀疏 TC 层用 NVIDIA 的稀疏 Tensor Core (GPU 里专门算矩阵乘的硬件); (2) Slot-Filling 层做「差分距离压缩」+ 低开销片上解码; (3) Residual 层保证计算结果精确。它是第一个在 50% 中等稀疏度下超越稠密矩阵乘法的 GPU 实现。

📖 专业版问题讲解 (展开)
Proposes a three-layer matrix storage format combining (i) Sparse-TC layer using NVIDIA sparse tensor cores, (ii) Slot-Filling layer with parallel differential distance compression and low-cost on-chip decoding, (iii) lightweight Residual layer ensuring correctness. Designs a SpMM kernel jointly using sparse tensor cores and CUDA cores with overlapped memory access.
⚙️ 方法：Sparse-TC + Slot-Filling + Residual three-layer storage format + SpMM kernel co-using sparse tensor cores and CUDA cores with overlapped compute-memory pipeline.
⚖️ 取舍：Limited to ~50% sparsity (higher sparsity reduces accuracy); requires NVIDIA sparse tensor core support (Hopper+); Slot-Filling decoding adds on-chip complexity.
📊 效果：First GPU kernel that outperforms dense matrix multiplication at ~50% moderate unstructured sparsity for LLM inference.
🏢 团队：Tao Lu, Haoyu Wang, Zonghui Wang, Keshen Xiang, Jiaheng Zhang, Wenzhi Chen → National University of Singapore; Zhejiang University
