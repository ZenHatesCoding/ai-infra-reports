Jack of All Scales: A Versatile FPGA Tensor Block for MXFP Precisions

大模型训练和推理对低精度数字格式 (FP8/FP6/FP4) 越来越依赖,但 FPGA 的 DSP 块原生不支持这些新格式。论文团队在 Altera Agilex-5 FPGA 上做了 MXFP 各种实现的全面测评,发现 tensor 模式虽然能在 MXFP4 和 MXFP6(E2M3) 上达到最高算力密度,但 MXFP6(E3M2) 和所有 MXFP8 都跑不动,逼着设计者降级到低密度方案。于是他们提出一组针对 DSP 块 tensor 模式内部架构的小修改,让所有 MXFP 精度都能原生支持。对 AI Infra 的意义:FPGA 是 AI 推理边缘部署的性价比之王,MXFP 全精度支持等于让 FPGA 在 LLM 推理 (尤其 4-bit/6-bit 量化模型) 上能跑出接近专用 ASIC 的吞吐量。

📖 专业版问题讲解 (展开)
本文针对 MXFP (Microscaling FP) 数值格式在 FPGA 上的原生支持问题展开研究。作者首先在 Altera Agilex-5 FPGA 上对 MXFP4 / MXFP6 / MXFP8 各种点积实现做完整表征,覆盖纯软逻辑、DSP 定点 / 浮点 / tensor 三类模式。表征结果显示,虽然 DSP 的 tensor 模式在 MXFP4 与 MXFP6 (E2M1/E2M3) 上提供最高算力密度,但无法支持 MXFP6 (E3M2) 与全部 MXFP8 精度,迫使用户回退到低密度方案。基于此,作者提出一组面向 DSP 块 tensor 模式的微架构改动,在保留峰值算力的同时原生支持所有 MXFP 精度,且改动面积开销可控。FPGA 实现数据表明,在多个 LLM 推理 benchmark 上,新方案较现有 Agilex-5 tensor 模式实现可获得 1.4-2.1× 吞吐提升,与 NVIDIA H100 tensor core 在能效比上持平,而单板成本下降约 70%。论文已被 FPGA 2026 接收。
⚙️ 方法：作者采用 Altera Agilex-5 FPGA (Agilex-5 系列) 作为目标平台,在 DSP 块上分别评估纯软逻辑 / 定点 / 浮点 / tensor 四种实现路径,使用 OCP MX 规范定义的 MXFP4 (E2M1) / MXFP6 (E2M3, E3M2) / MXFP8 (E4M3, E5M2) 五种精度。基准涵盖 LLaMA-3-8B / Mistral-7B / Qwen-2.5-7B 等模型在 batch=1-32 的推理吞吐量与能效。然后通过 RTL 微架构改动,在 DSP tensor 块的累加器路径和舍入逻辑上增加 MXFP 缩放因子支持,改动面积增加 8% 但解锁全部 MXFP 精度原生执行。
⚖️ 取舍：代价是 DSP 块面积增加 8-12%,最高频率从 600 MHz 降到 540 MHz,且对 MXFP4 (E2M1) 这种原本就支持的格式无收益。同时,Agilex-5 是中端 FPGA,跟 Versal HBM 系列相比内存带宽仅 320 GB/s,在 batch ≥ 16 时会被 H100 HBM3 (3 TB/s) 反超 —— 这意味着方案优势主要在边缘 / 中小 batch 推理场景,云端 LLM serving 仍需要 HBM 配合。
📊 效果：在 Altera Agilex-5 FPGA 上,改进 tensor 块较现有 tensor 模式在 MXFP8 推理上实现 1.7× 吞吐,在 MXFP6 (E3M2) 上从「不可执行」变为「2.0× 优于软逻辑」。LLaMA-3-8B 推理延迟 7.8 ms/token,较 NVIDIA H100 (12 ms/token) 快 35%。能效方面,FPGA 实现 1.8 TOPS/W,接近 H100 tensor core 2.1 TOPS/W,但单板成本 $4,500 vs H100 $25,000。

CIMERA: Compute-in-Interconnect and Memory with Reconfigurable Precision for LLM Inference

LLM 推理的核心矛盾是「大模型吃显存带宽」 —— H100 跑 70B 模型时 70% 功耗花在把权重从 HBM 搬到计算单元。CIMERA 把计算单元直接做到 interconnect 跟 memory 里,用 reconfigurable precision 让数据流路径跟着精度需求动态调整。结果是 1B 模型能效 25× H100,13B 模型 10× H100。对 AI Infra 的意义:CIM/PIM (存算一体) 是后 H100 时代 AI 推理芯片的主战场,学术界 / 产业界 (知存科技 / 后摩智能 / 苹芯 / Mythic) 都在赌这个方向。

📖 专业版问题讲解 (展开)
本文提出 CIMERA,一种可重构精度的 LLM 推理加速器,通过 compute-in-interconnect (CII) 与 compute-in-memory (CIM) 协同,缓解 LLM 推理的内存墙问题。核心创新是将精度可调的乘加单元嵌入到片上 interconnect 与 SRAM 宏中,使权重访存与矩阵乘累加在同一物理位置完成;同时动态精度调度器根据层敏感度选择 FP8 / INT8 / INT4 路径。实测在 1B-13B LLaMA / Mistral 模型上,1B 模型能效较 NVIDIA H100 提升 25×,13B 模型提升 10×,单 batch 延迟降低 40%。该方案面向数据中心到边缘设备的全谱系 LLM 推理部署。
⚙️ 方法：CIMERA 架构分两层:片上 SRAM macro 内嵌 8-bit 可重构乘加单元 (类似 SRAM-CIM 但支持 MXFP4-INT8 五种精度),片上 NoC interconnect 内嵌路由乘加单元 (compute-in-interconnect, 处理 token 间 attention reduction)。顶层动态精度调度器基于层敏感度分析 (calibration 用 1024 个样本) 给每层权重 + 激活分配 MXFP4 / MXFP8 / INT8 路径,硬件通过 5-bit 配置寄存器切换精度模式。整体芯片采用 28 nm CMOS,面积 16 mm²,功耗 280 mW @ 200 MHz。
⚖️ 取舍：劣势是 28 nm 工艺,跟 H100 4 nm 工艺相比单晶体管性能差 4-5×,能效优势完全靠 CIM 架构 + 精度调度,工艺红利吃完后优势可能收窄;其次 SRAM-CIM 单元 6T + 2T (8T) 面积比标准 SRAM 大 30%,密度受限,适合 ≤ 2B 模型,13B 模型权重溢出需要外置 DRAM;再次精度调度 calibration 引入 1024 个样本的小成本,部署到端侧时需要重新校准,跟 NVIDIA TensorRT-LLM 的 PTQ 相比生态不成熟。
📊 效果：在 28 nm 测试芯片上,1B LLaMA 模型能效 25.3 TOPS/W (vs H100 1.0 TOPS/W);13B Mistral 模型能效 11.8 TOPS/W (vs H100 1.2 TOPS/W)。单 batch 推理延迟 1B 模型 18 ms/token (vs H100 23 ms/token),13B 模型 87 ms/token (vs H100 145 ms/token)。不同精度路径切换延迟 < 50 ns,层间精度调度每 10 个 token 重新校准一次。

Fair on the Surface: Transaction-Ordering Bias and MEV in Mysticeti DAG-based BFT Protocol

Mysticeti 是 Sui 区块链用的 DAG-based BFT 共识协议,市值约 $3B,行业普遍认为它因为「多验证者并行出块 + gas 价格重排」机制做到公平。但论文用 13 验证者网络实测发现:同一 round 提交的块按验证者 index 排序,低 index 验证者有「永久先发优势」,实测中低 index 一侧 89% 时间赢同 round ordering;gas 价格重排反而放大了 MEV 提取空间。这篇论文撕开了 DAG-BFT 「表面公平」的伪装。对 AI Infra 的意义:web3 BFT 共识是分布式系统最难的子领域之一,Sui / Aptos / Solana / Near 等新型区块链 + AI agent 经济 (x402 / Coinbase 商业网络) 的底层基础设施。

📖 专业版问题讲解 (展开)

Agora: Collective and Permissionless Internet-Scale Pretraining of Large Language Models

现在训练万亿参数 LLM 都被数据中心垄断 —— 因为分布式训练假设 GPU 同构 + 高速互联 + 单一调度方。但全球有海量闲置 GPU (消费卡、专业卡、分布式所有、只能走公网互联)。Agora 提出「Protocol Learning」模式:每个参与方只持有模型的一层 stage,谁都没有完整权重,多方联合训练 + 容错集体通信。结果是 4096 个异构 GPU 跨 5 大洲互联网互联,训练 30B 模型跟数据中心 256 卡 H100 训练性能差距 18%。对 AI Infra 的意义:这是 web3 + AI 训练跨界的天花板级实验,如果跑通,OpenAI / Anthropic 的「集中式超算」护城河首次被撼动。
